Visão Geral

O sistema WOSA Reports é uma solução completa para processamento e análise de relatórios JSON relacionados a tópicos Kafka. O sistema permite:

Upload de Arquivos

Interface drag-and-drop para upload de arquivos JSON com validação automática

Armazenamento

Persistência completa dos dados em banco de dados com histórico de mudanças

Análise

Processamento automático de estatísticas e detecção de problemas

Relatórios

Geração automática de 10 tipos diferentes de relatórios de análise

Formato JSON

O sistema espera arquivos JSON no seguinte formato:

{ "created_at": "2025-01-17T08:55:39.690046", "topics": [ { "name": "prv-ferren-debeizium-status", "object": { "bridged_topic": null, "consumers": [], "missing_consumers": [], "missing_producers": ["ferren"], "name": "prv-ferren-debeizium-status", "producers": [], "stats": { "average_message_size": 0, "cleanup_policy": "compact", "estimated_size": 0, "last_message_date": "0001-01-01T00:00:00Z", "last_stat_retrieval_date": "2025-01-17T08:47:42.926Z", "maximum_message_size": 0, "messages_last_30d": 0, "minimum_message_size": 0, "partition_number": 5, "replication_factor": 3, "retention": "66960000000", "total_messages": 0 } } } ] }

Estrutura dos Dados

  • created_at: Data de criação do relatório (ISO 8601)
  • topics: Array de objetos de tópicos
  • name: Nome do tópico
  • object: Objeto contendo dados do tópico
  • producers/consumers: Arrays de produtores e consumidores
  • missing_producers/consumers: Arrays de produtores/consumidores ausentes
  • stats: Estatísticas do tópico

API Endpoints

Upload e Processamento

POST /api/v1/wosa/upload
Upload de arquivo JSON WOSA
POST /api/v1/wosa/process/{file_id}
Processar arquivo específico

Consulta de Dados

GET /api/v1/wosa/files
Listar arquivos processados
GET /api/v1/wosa/topics
Listar tópicos com filtros
GET /api/v1/wosa/topics/{topic_id}
Detalhes de tópico específico

Relatórios

POST /api/v1/wosa/reports
Gerar relatório específico
GET /api/v1/wosa/reports
Listar relatórios gerados

Utilitários

POST /api/v1/wosa/generate-id
Gerar próximo ID disponível

Estrutura do Banco de Dados

Tabelas Principais

  • files: Armazenamento de arquivos JSON com metadados
  • topics: Tópicos Kafka com estatísticas completas
  • topic_producers: Produtores de tópicos
  • topic_consumers: Consumidores de tópicos
  • missing_producers: Produtores ausentes
  • missing_consumers: Consumidores ausentes
  • topic_history: Histórico de mudanças
  • reports: Relatórios gerados
  • report_items: Itens dos relatórios

Características Especiais

  • Detecção automática de ambiente: dev/prod/e2e baseado no nome
  • Tracking de histórico: Primeira e última aparição de tópicos
  • Status de arquivos: pending/processing/completed/error
  • Nomes únicos: Arquivos com timestamp em minúsculo
  • Formato de data: YYYY-MM-DD HH:mm:SS

Como Começar

1. Iniciar o Sistema

uvicorn app.main:app --reload

2. Acessar a Interface

  • Interface WOSA: http://localhost:8000/wosa
  • Documentação da API: http://localhost:8000/docs
  • Interface Principal: http://localhost:8000/

3. Upload de Arquivo

  1. Acesse a interface WOSA
  2. Faça upload do arquivo JSON
  3. O sistema processará automaticamente
  4. Visualize os resultados e gere relatórios

4. Exemplo de Uso via API

# Upload de arquivo curl -X POST "http://localhost:8000/api/v1/wosa/upload" \ -F "file=@sample_wosa_data.json" \ -F "user_id=test_user" # Processar arquivo curl -X POST "http://localhost:8000/api/v1/wosa/process/1" # Gerar relatório curl -X POST "http://localhost:8000/api/v1/wosa/reports" \ -H "Content-Type: application/json" \ -d '{"report_type": 1, "generated_by": "test_user"}'
